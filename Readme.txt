
版本二：
更新票务管理系统，较上次加入链表存储，文件操作，进行不同源文件实现各自功能，更加清晰的工程架构。
1、首先显示选择界面：1、管理员 2、乘客，3、退出。根据选择进入相应的界面；
2、然后进行登录和注册操作：首次登录先进行注册，进入注册界面；非首次登录，进行输入账号和密码。
   参考https://docs.pingcode.com/baike/962406。
3、如果选择1、管理员：进入的界面显示：1、增加航班 2、删除航班 3、显示航班的订票信息 4、修改航班信息 5、退出
                                                   
            航班信息的结构体：{航班号，出发地，目的地，飞行日期，出发时间，到达时间，机型，票价，乘客定额，余票}
            乘客信息结构体：   {预定航班号，乘客姓名，身份证号，座位等级，出发地，目的地，飞行日期，出发时间}
            管理员的4：修改航班信息：
                                  修改航班的起飞时间；
                                  修改航班的降落时间；
                                  修改航班的起飞城市；
                                  修改航班的降落城市；
                                  修改航班的机票价格；}
4、如果选择2、乘客：进入界面显示：1、查询航线信息 2、订票 3、退票 4、查询订票信息 5、退出
            1、查询航线信息： 按航班号查询、按终点站查询、按起点站查询、返回
